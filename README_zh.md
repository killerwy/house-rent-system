## 📝 项目介绍
**house-rent-system** 是一套面向中小型房产中介打造的完整租房管理系统。系统覆盖房源库存、房东租客档案、租赁合同、费用收缴、员工权限管控、数据备份恢复全流程数字化管理。

后端采用轻量化Flask框架开发，前端使用原生页面搭建，整体分层架构清晰；业务底层通过数据库触发器、存储过程实现数据联动；内置三级角色权限体系，标准化中介日常租赁业务流程，提升门店管理效率。

## ✨ 核心功能
### 基于角色的权限管控
- 内置三类角色：超级管理员、中介员工、只读员工
- JWT令牌登录认证，接口级权限校验
- 员工账号管理：新增、编辑、重置密码、删除
- 存在业务操作记录的账号受删除保护，防止误删

### 房源公寓管理
- 房源全增删改查，支持面积、租金、配套设施、房屋状态等完整属性录入
- 户型分类独立管理
- 省市区三级联动地址选择，自动地址去重
- 多维度筛选：地区、房源状态、户型、关键词搜索
- 房源状态与租赁合同自动同步更新

### 房东&租客信息管理
- 完整房东、租客档案维护
- 手机号、身份证号格式校验
- 手机号、身份证信息唯一性校验，禁止重复录入
- 绑定有效房源/合同的客户记录限制删除

### 租赁合同管理
- 入住登记：自动校验房源是否可出租
- 数据库触发器自动同步更新房源出租状态
- 退租登记：记录实际退房日期并重置房源状态
- 合同列表支持关键词检索、状态筛选
- 每条合同留存操作人记录

### 费用收缴管理
- 支持三类收费：租金、押金、中介费
- 所有收费记录与租赁合同强关联
- 分页查询，支持收费类型筛选、关键词检索
- 新增/终止合同时，触发器自动生成对应费用单据

### 数据统计可视化
- 通过存储过程按户型统计房源租赁情况
- ECharts柱状图直观展示已租/空置房源数量对比
- 自动计算各户型出租率

### 数据库备份与恢复
- 一键MySQL数据库备份，文件自动添加时间戳命名
- 备份文件列表与历史记录管理
- 路径安全防护，防止非法路径越权恢复
- 仅超级管理员可操作备份恢复功能

### 通用工具能力
- 统一标准化API返回格式
- 全局分页工具类，所有列表查询复用
- 房源信息聚合数据库视图，快速汇总房源数据
- 前端基于Bootstrap搭建，界面简洁清爽

## 🎯 适用场景
- **中介门店日常运营**：统一平台数字化管理房源、客户、租赁合同
- **员工分级权限管控**：给店长、一线业务员、数据审核员分配差异化操作权限
- **业务数据分析**：通过内置统计功能查看出租业绩、空置率、房源结构
- **数据安全保障**：定期数据库备份，防止数据丢失，支持故障回滚
- **中小房产中介门店**：开箱即用的租赁管理方案，部署与维护成本低

## ⚡ 快速部署
### 1. 克隆项目仓库
```bash
git clone https://github.com/killerwy/house-rent-system.git
cd house-rent-system
```

### 2. 初始化数据库
执行脚本 `backend/db/init_sql.sql`，自动创建名为 `house_rent` 的数据库并导入测试演示数据

### 3. 环境变量配置
复制环境变量模板文件
```bash
# Windows系统
copy .env.example .env
# macOS / Linux系统
cp .env.example .env
```
根据本地MySQL账号信息修改 `.env` 文件内数据库配置

### 4. 安装后端依赖并启动服务
进入后端目录
```bash
cd backend
```
创建并激活虚拟环境
```bash
# Windows
python -m venv venv
venv\Scripts\activate
# macOS/Linux
python -m venv venv && source venv/bin/activate
```
安装依赖包：
```bash
pip install -r requirements.txt
```
启动后端服务
```bash
python app.py
```
服务默认运行地址：`http://localhost:5000`

### 5. 访问系统
使用浏览器打开 `frontend/index.html`
> 默认超级管理员账号：`admin` / 密码：`123456`

## 📖 使用说明
本章节提供所有核心模块分步操作指南

### 1：系统登录
1. 浏览器打开 `frontend/index.html`
2. 在登录表单输入账号密码
3. 点击登录向后端接口发送鉴权请求
4. 验证通过后，JWT令牌存入本地存储，自动跳转系统首页仪表盘
5. 令牌过期或未登录直接访问页面时，系统自动重定向至登录页

### 2：户型分类管理
1. 菜单路径：房源管理 → 户型管理
2. 新增户型：
   - 点击新增按钮弹出编辑弹窗
   - 填写户型名称、备注信息
   - 提交保存至数据库
3. 编辑户型：
   - 对应数据行点击编辑按钮
   - 弹窗修改内容后提交保存
4. 删除户型：
   - 点击删除并确认弹窗提示
   - 若该户型已绑定房源，系统禁止删除

### 3：房源信息管理
1. 菜单路径：房源管理 → 房源列表
2. 房源筛选：
   - 下拉框选择省、市、区三级联动地址
   - 筛选户型、房源出租状态
   - 输入地址/房东关键词模糊检索，执行筛选
3. 新增房源：
   - 点击新增房源打开弹窗
   - 填写必填项：地址、地区、户型、房东、面积、月租金、配套设施、房源状态
   - 提交表单，系统自动校验地址重复
4. 编辑房源：对应行点击编辑，弹窗修改信息后保存
5. 删除房源：点击删除并确认，存在关联租赁合同时禁止删除

### 4：房东信息管理
1. 菜单路径：客户管理 → 房东列表
2. 搜索房东：输入姓名/手机号关键词检索
3. 新增房东：
   - 点击新增，填写姓名、手机号、身份证号、居住地址
   - 系统自动校验手机号、身份证格式与唯一性
   - 提交创建房东档案
4. 编辑房东：点击编辑按钮修改信息并保存
5. 删除房东：确认弹窗后执行删除；名下存在房源时无法删除

### 5：租客信息管理
1. 菜单路径：客户管理 → 租客列表
2. 通过姓名、手机号、身份证号关键词检索租客
3. 新增租客：
   - 点击新增，填写姓名、手机、身份证、工作单位
   - 前后端双重校验手机号、身份证格式与重复
4. 修改租客信息：编辑弹窗更新内容后提交
5. 删除租客：存在在租合同的租客禁止删除

### 6：租赁合同创建与管理
#### 6.1 新建租赁合约（入住登记）
1. 菜单路径：租赁业务 → 新建合同
2. 下拉框仅展示状态为空置的房源供选择
3. 选择对应租客信息
4. 填写租赁起止日期、实际月租金
5. 提交完成签约登记
6. 数据库触发器自动执行：
   - 将房源状态改为已出租
   - 自动生成租金、押金、中介费收费记录

#### 6.2 查看全部合同
1. 菜单路径：租赁业务 → 合同列表
2. 关键词检索或按合同状态（在租/已退租）筛选
3. 点击详情弹窗查看完整合同信息

#### 6.3 房源退租办理
1. 在合同列表找到状态为「在租」的合约
2. 点击退租按钮打开退房弹窗
3. 填写实际退房日期
4. 提交退租表单
5. 数据库触发器自动执行：
   - 合同状态标记为已退租
   - 房源状态恢复空置
   - 自动生成押金退还收费记录

### 7：收费记录管理
1. 菜单路径：费用管理 → 收费记录列表
2. 筛选条件：合同关键词、费用类型（租金/押金/中介费）
3. 手动新增额外收费单据：
   - 点击新增收费
   - 选择对应租赁合同，选择费用类型，填写金额与备注
   - 提交保存单据

### 8：租赁数据统计图表
1. 菜单路径：数据统计 → 户型出租统计
2. 页面自动读取数据库存储过程统计数据
3. 表格展示：户型名称、总房源数、已租数量、空置数量、出租率
4. 柱状图可视化对比各户型已租/空置房源数量
5. 浏览器窗口缩放时图表自动自适应宽度

### 9：超级管理员专属功能
#### 9.1 员工账号管理
1. 菜单路径：系统管理 → 员工管理
2. 按账号、真实姓名关键词检索员工
3. 新增员工：填写账号、密码、姓名、手机、身份证，分配角色权限
4. 编辑员工：修改姓名、联系方式、身份、权限（用户名不可修改）
5. 重置员工密码：点击重置密码，输入新密码提交
6. 删除员工账号：
   - 超级管理员账号禁止删除
   - 存在历史业务操作记录的员工无法删除

#### 9.2 数据库备份与恢复
1. 菜单路径：系统管理 → 数据备份
2. 一键备份：点击创建备份，系统生成带时间戳的SQL备份文件并刷新列表
3. 查看备份历史：展示所有.sql备份文件及创建时间
4. 从备份恢复数据库：
   - 对应备份文件点击恢复按钮
   - 两次弹窗确认警告后开始恢复
   - 系统将使用备份文件覆盖当前数据库全部数据

### 10：全局房源总览视图
1. 菜单路径：数据视图 → 房源总览
2. 多维度筛选：省市区、户型、房源状态、关键词检索
3. 页面聚合展示房源地址、房东联系方式、户型、租金、出租状态，无需切换多页面查看
